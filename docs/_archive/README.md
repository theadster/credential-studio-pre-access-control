# Documentation Archive

This directory contains archived documentation files that are no longer actively maintained but preserved for historical reference and knowledge retention.

## Why Files Are Archived

- **Resolved:** Bug fixes that have been applied and verified
- **Superseded:** Replaced by newer, more comprehensive documentation
- **Historical:** Records of completed projects, phases, or migrations
- **Outdated:** Information that is no longer relevant to current codebase

## Archive Organization

```
_archive/
├── fixes/              # Resolved bug fixes and investigations (380+ files)
├── migrations/         # Completed migration runbooks (7 files)
├── guides/             # Superseded implementation guides (37 files)
├── testing/            # Completed test documentation (23 files)
├── enhancements/       # Superseded feature documentation (6 files)
└── misc/               # Resolved miscellaneous items (7 files)
```

## Searching the Archive

All archived files remain searchable. To find archived documentation:

1. **By topic:** Search within the relevant subdirectory (e.g., `_archive/fixes/`)
2. **By keyword:** Use full-text search across all archive files
3. **By date:** Files are organized by category; check git history for timestamps

## Restoring Files

If you need to reference or restore an archived file:

1. Check the appropriate subdirectory for your topic
2. Review the file to understand historical context
3. If the information is still relevant:
   - Move it back to the main `docs/` directory
   - Update the file to reflect current state
   - Add frontmatter metadata (see `docs/README.md`)
4. Update any cross-references
